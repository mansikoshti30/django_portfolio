# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Activate virtual environment (required before any command)
source venv/Scripts/activate          # Windows Git Bash / MSYS2
# or
venv\Scripts\activate                 # Windows CMD/PowerShell

# Run development server
python manage.py runserver

# Database migrations
python manage.py makemigrations
python manage.py migrate

# Create superuser
python manage.py createsuperuser

# Run all tests
python manage.py test

# Run tests for a single app
python manage.py test core

# Run a specific test case or method
python manage.py test core.tests.HomeViewTest
python manage.py test core.tests.HomeViewTest.test_status_code

# Collect static files (production)
python manage.py collectstatic
```

## Architecture

**Project layout:**
- `config/` — Django project package: `settings.py`, `urls.py`, `wsgi.py`, `asgi.py`
- `core/` — primary app; add further apps at the same level as `core/`
- `templates/` — project-wide templates; app-specific templates go in `templates/<app_name>/`
- `static/` — project-wide static files (`css/`, `js/`, `images/`)
- `staticfiles/` — generated by `collectstatic`, not committed
- `media/` — user-uploaded files, not committed

**Settings:** `config/settings.py` reads `DJANGO_SECRET_KEY`, `DJANGO_DEBUG`, and `DJANGO_ALLOWED_HOSTS` from environment variables with sensible development defaults.

**URL routing:** `config/urls.py` is the root router. Each app exposes its own `urls.py` and is included with a namespace (`app_name`). Use `{% url 'core:home' %}` style in templates.

**Templates:** `base.html` at the project root provides the site shell (nav, messages, footer, static asset tags). App templates extend it with `{% extends 'base.html' %}`.

**Adding a new app:**
1. `python manage.py startapp <name>`
2. Add `'<name>'` to `INSTALLED_APPS` in `config/settings.py`
3. Create `<name>/urls.py` with `app_name = '<name>'`
4. Include it in `config/urls.py` with `path('<prefix>/', include('<name>.urls'))`
5. Add templates under `templates/<name>/`
